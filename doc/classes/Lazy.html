Here‘s the problem. When we do this: 

  x,y = z

…Ruby calls z.respond_to?(to_a) to see if z is an array. If it is, it will do the multiple assignment; if not, it will just assign x=z and set y=nil.

We want our Lazy to forward the respond_to? call to our fibo list. But it doesn‘t forward it, because we used the method_missing to do the proxying — and every object implements respond_to? by default, so the method isn‘t missing! The respond_to? doesn‘t get forwarded; instead, out Lazy says "No, I don‘t respond to to_a; thanks for asking." The immediate solution is to forward respond_to? manually:
